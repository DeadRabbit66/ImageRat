───────────────────────────────────────────────────────────────

🐀 BURY text into image:

   1. Введите текст в поле
   2. Нажмите «BURY text into image»
   3. Выберите изображение (PNG, JPG, BMP)
   4. Укажите место для сохранения

───────────────────────────────────────────────────────────────

🐀 DIG from image:

   1. Нажмите «DIG from image»
   2. Выберите картинку со скрытым текстом
   3. Текст появится в поле

   💡 ИЛИ просто перетащите картинку в окно программы!

───────────────────────────────────────────────────────────────

⚠️ ВАЖНЫЕ ОСОБЕННОСТИ:

   • JPEG/JPG автоматически конвертируются в PNG
   • Оригинальные файлы НЕ изменяются
   • Сохранённая картинка получает суффикс _enc
   • Текст НЕ шифруется (только LSB)
ВНИМАНИЕ! Если канал передачи изображения модифицирует фаил, текст может быть утерян. Для уверенности прячьте картинку в .zip перед передачей!

───────────────────────────────────────────────────────────────

🔐 ДЛЯ КРИПТОГРАФИЧЕСКОЙ СТОЙКОСТИ:

Сообщение никак не шифруется, просто вставляется в картинку по алгоритму LSB. Если вы хотите криптографическую стойкость - шифруйте предварительно сообщение с помошью Cryptic Wytch.

Simple steganography tool.
For full security, use together with Cryptic Wytch.

🔗 Cryptic Wytch: https://github.com/DeadRabbit66/CrypticWytch

───────────────────────────────────────────────────────────────

🐀 BURY text into image:

   1. Enter your text in the field
   2. Click «BURY text into image»
   3. Choose an image (PNG, JPG, BMP)
   4. Select where to save the result

───────────────────────────────────────────────────────────────

🐀 DIG from image:

   1. Click «DIG from image»
   2. Choose an image with hidden text
   3. The text will appear in the field

   💡 OR simply drag & drop the image into the program window!

───────────────────────────────────────────────────────────────

⚠️ IMPORTANT NOTES:

   • JPEG/JPG are automatically converted to PNG
   • Original files are NOT modified
   • Saved image gets the suffix _enc
   • Text is NOT encrypted (LSB only)
ATTENTION! If the image transfer channel modifies the file, the text may be lost. To be on the safe side, zip the image before transferring!

───────────────────────────────────────────────────────────────

🔐 FOR CRYPTOGRAPHIC SECURITY:

   The message is not encrypted — it's just embedded via LSB algorithm.
   If you need cryptographic strength, encrypt your message first using Cryptic Wytch.
